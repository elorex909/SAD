Chapter 1: Definition of Systems & SAD

Understanding the building blocks of systems analysis.

What is a System?

A system is a collection of interconnected elements—such as people, processes, technology, or resources—that work together to achieve a specific purpose or function. These elements interact within a defined boundary and are often influenced by feedback to produce outcomes aligned with the system's goal.

In the context of Systems Analysis and Design (SAD), a system most often refers to an information system: a set of hardware, software, data, people, and procedures that interact to collect, process, store, and distribute information to support organizational functions.

Key Features of Any System

Components: The individual parts (e.g., hardware, employees, rules).
Interactions: The relationships and data flow between components.
Purpose: The system’s intended objective or goal.
Boundary: The line defining what is inside the system versus its external environment.
Feedback: Information loops that influence behavior.
Examples: Ecosystem, Computer System, Organizational System.

Key Components of SAD

1. Systems Analysis

Focuses on understanding the problem and defining requirements. Key activities include:

  • Requirement Gathering
  • Feasibility Analysis
  • Problem Identification
  • Modeling (e.g., DFDs, ERDs)

2. Systems Design

Focuses on creating a blueprint for the system. Key activities include:

  • Architectural Design
  • Interface Design (UI/UX)
  • Data Design
  • Prototyping

Core Components of a System

Inputs: Resources or data that enter the system (e.g., work hours).
Processes: Transformations that manipulate inputs (e.g., calculating wages).
Outputs: The results generated by the system (e.g., paychecks).
Feedback: Information about outputs used to adjust the system.
Environment: External factors outside the system's control (e.g., tax laws).
Controls: Mechanisms to ensure correct and secure operation.

Chapter 2: Types of Systems

Classifying systems based on their function, nature, and behavior.

Key Types of Information Systems

Transaction Processing (TPS)

  • Purpose: Handles routine, high-volume, day-to-day operational data.
  • Characteristics: High reliability, speed, and accuracy.
  • Examples: Point of Sale (POS), Banking System.

Management Information (MIS)

  • Purpose: Provides managers with summarized reports for routine decision-making.
  • Characteristics: Focuses on structured data and predefined reports.
  • Examples: Monthly Sales Dashboard, Inventory Reports.

Decision Support (DSS)

  • Purpose: Assists in solving complex, semi-structured problems using data analysis.
  • Characteristics: Interactive, "what-if" analysis, data visualization.
  • Examples: Financial Forecasting, Supply Chain Optimization.

Executive Information (EIS)

  • Purpose: Delivers high-level KPIs and external data for senior executives' strategic planning.
  • Characteristics: User-friendly, high-level summaries.
  • Examples: BI Dashboard (Market Trends).

Expert Systems

  • Purpose: Emulates the decision-making ability of a human expert using AI.
  • Characteristics: Uses a knowledge base and inference rules.
  • Examples: Medical Diagnosis System.

Enterprise Resource (ERP)

  • Purpose: Integrates and manages all core business processes (finance, HR, etc.).
  • Characteristics: Cross-functional, scalable, unified data.
  • Examples: SAP, Oracle ERP.
